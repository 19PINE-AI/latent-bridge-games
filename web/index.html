Latent Bridge

A learned continuous latent channel between a frozen 9 B fast model and a frozen 8 B reasoning model — beating the text-bridge default on 6 of 7 Atari games.

The thesis in one minute

Real-time interactive AI is structurally torn: reasoning models (Claude, GPT-5, Qwen-Thinking) deliberate well but tick too slowly for 60 Hz environments; streaming models hit the tick rate but lack reasoning depth. Production splits the workload by routing text prompts between the two, which is bandwidth-limited — text serializes hundreds of bits per call where a continuous latent channel could carry hundreds of thousands.

We test this on Atari: a frozen 9 B fast model (MiniCPM-o 4.5) at 15 Hz, a frozen 8 B slow model (Qwen3-VL-8B-Thinking) at 1 Hz, coupled via a trainable 33 M-param latent bridge prepended LLaVA-style to the fast model's input embedding.

Headline result — Road Runner (+ infinite, the cleanest demo)

🏆 Road Runner — F = 0 L = 967

The fast-only agent can't commit to any direction and scores 0. The slow model says "run right to escape the Coyote" and the latent bridge unlocks coherent rightward escape. This is fast/slow coupling in its purest visible form — the bridge isn't just "adding a prior"; it's unlocking a behavior the action head couldn't produce alone.

Refined headline

When latent wins (6 games)

Slow-model content is continuous-rich — coordinates, distances, fuel levels, multi-entity spatial state. Text serializes lossy; latent preserves the structure.

Mean L−T gap: +44 % across 6 wins, from +18 % (Enduro) to +82 % (River Raid robust).

When text wins (1 game)

Slow-model content is categorical — "jump UP-RIGHT to tile 3,2." Discrete decisions compress losslessly into text; the latent's continuous compression introduces noise.

Q*bert: T=125, L=50. This refines the bandwidth claim, doesn't refute it.

The Stage A OOD diagnosis

Initial pattern: L > T cleanly on MsPacman/Seaquest/RoadRunner. But L = T = 0 on Space Invaders despite three knob-tuning attempts (random-T, expert-T, aggressive-prompt). The bridge wasn't broken.

Diagnosis: Stage A behavioral cloning was trained on bare prompts. At deployment T appends a text suffix and L prepends bridge tokens — both present out-of-distribution input to the frozen action head. On reward-symmetric games the resulting policy drift still scores; on reward-asymmetric games (FIRE-only scoring) it collapses to zero.

Fix recipe: retrain Stage A with --suffix-prob=0.5 (mixed bare/suffix-augmented prompts). Validated on two games: Space Invaders broke its zero floor (0 → 15/18); River Raid flipped from collapse to +82 % L > T, the largest gap in our sweep.

Targeted, not universal: applying robust Stage A to games that already work (MsPacman, Seaquest) actually hurt them. So the recommendation is: use robust Stage A only for games that show OOD collapse, not as a default.

Architecture

slow model (Qwen3-VL-8B-Thinking, 1 Hz)
   └─ residuals at layer 24, last N=8 token positions (4096-d)
       └─ ThoughtProjection: 4096 → 4096 → 4096 + LayerNorm (33.6 M trainable)
            └─ N=8 latent tokens in fast model's embedding space
                 └─ PREPENDED to fast model's input sequence
                      └─ fast model (MiniCPM-o 4.5, 15 Hz) — all 36 LLM layers attend
                           └─ action_head on last hidden state → action logits

Trainable: only the slow model's ThoughtProjection (33.6 M params). Both base models entirely frozen.

Loss: KL(student ‖ teacher), where the teacher is T (text-bridge runtime) and the student is L (latent-bridge runtime). The bridge learns to make the latent prepend reproduce the text-channel's action distribution.

Bandwidth: empirically N = 8 tokens is the Goldilocks sweet spot. N = 4 → L drops to 296; N = 16 → L drops to 259 (worse than fast-only).

Ordering: S < F < T < L. The slow model alone deliberates competently but can't react fast enough; the fast model alone reacts well but lacks strategic context; bridging the two — especially via latent — beats either single model.

What this project produced

  • 9-game empirical sweep validating the bandwidth thesis (with one categorical exception that refines, not refutes)
  • OOD-brittleness diagnosis for action heads under bridge prompts, with a recipe that fixes 2 of 2 collapsed games tested
  • v2 LLaVA-style bridge architecture (after v1 cross-attn at 2 layers failed); 33 M trainable params, both base models frozen
  • Stage D PPO driver (works but needs tuning; mode-collapsed at update 7 on first run)
